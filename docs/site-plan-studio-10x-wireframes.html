1. Draw or analyze

AI proposes shapes, but the editor treats them exactly like user-created annotations.

  • Normalized 0-1 geometry
  • Editable forever
  • Layer-aware

2. Bind to Property Record

Space labels, suite numbers, rates, and tenant logos reference stable record IDs.

  • Space.squareFootage
  • Tenant.logoAssetId
  • Occupancy.suiteNumber

3. Publish outputs

Flatten only when exporting a reusable marketing asset. The source plan stays editable.

  • Export PNG
  • Insert into flyer
  • Regenerate later

Inspector upgrade

Replace the long form with contextual sections: selection summary, binding, style presets, geometry, and danger zone.

Command palette

Add quick actions: "Bind selected to Suite 120", "Create SF label", "Accept high confidence suggestions".

Keyboard model

Figma-like shortcuts: V select, H pan, Z zoom, R rectangle, P polygon, L label, A analyze.

Compact studio layout

On smaller screens, side panels collapse into bottom sheets. The canvas, zoom HUD, and tool rail remain primary.

Panel resize behavior

  • Left panel: 220-420px
  • Right panel: 300-520px
  • Double click gutter to collapse
  • Persist per user in local storage

Visual direction

Glass-light panels over a cool drafting-table canvas. Accent orange only for availability and review attention.

Icon language

Use Lucide-style outline icons everywhere. No emojis. Toolbar icons get 48px touch targets; panel controls get 28px.

Canvas controls

Always-visible zoom HUD, pan mode, minimap, fit-to-screen, and keyboard hints in the status bar.

Information architecture

Left panel: pages/assets/suggestions. Right panel: inspector/layers/data. Center: canvas and ephemeral controls.

AI behavior

AI Analyze imports a layer, review mode focuses suggestions, and acceptance converts them into normal broker-edited annotations.
